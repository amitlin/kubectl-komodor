# Contributing to kubectl-komodor

Thank you for considering contributing!

## How to Contribute

- Fork the repo and create your branch from `main`.
- Ensure your code passes linting and formatting (see `.pre-commit-config.yaml`).
- Use [conventional commits](https://www.conventionalcommits.org/).
- Open a pull request and describe your changes.

## Code Style

- Use Prettier for formatting.
- Use ESLint for linting TypeScript/JavaScript.

## Running Checks

- Run hooks: `pipx run pre-commit run --all-files`

## Issues

If you find a bug or want a feature, open an issue first to discuss.
